========================================================================
プログラム仕様書：OBSニコニコ動画風コメント弾幕プラグイン (obs-niconico-danmaku)
========================================================================

本ドキュメントは、OBS Studioのシーン上にニコニコ動画風のコメント弾幕をオーバーレイ描画する入力ソースプラグイン「obs-niconico-danmaku」の機能・設計および動作仕様を定義したものです。他のAIが本プログラムの構造や振る舞いを即座に理解できるよう整理されています。

------------------------------------------------------------------------
1. プログラム概要
------------------------------------------------------------------------
■ 正式名称: obs-niconico-danmaku.dll
■ OBS上での表示名: ニコニコ風弾幕 (NicoNicoDanmaku)
■ プラグインタイプ: 入力ソース (OBS_SOURCE_TYPE_INPUT / 出力フラグ OBS_SOURCE_VIDEO)
■ 開発言語・規格: C++20, GDI+
■ 動作環境: Windows 10/11 (64bit), OBS Studio 31.1.x 以上

------------------------------------------------------------------------
2. 主要機能
------------------------------------------------------------------------
① コメント読み込み・流し表示:
   指定されたテキストファイル（UTF-8）からコメントを読み込み、画面上を左から右、または右から左へ滑らかにスクロール表示します。
② 重なり・追い越し防止（レーン衝突管理）:
   画面の縦幅をフォントサイズに応じた複数の「レーン」に分割。先行コメントと新コメントの速度差から画面内での追いつき（衝突）を物理予測し、重ならない安全なレーンを自動選択して投入します。
③ 2つの監視モード:
   - ループ再生（デモモード）: ファイルに書かれた全テキスト行からランダムに繰り返しコメントを流します。
   - リアルタイム追記監視: テキストファイルの更新日時を監視し、末尾に新しく追記された行のみを即座に画面へ流します。
④ 賑やかし自動混入:
   設定された確率に基づき、ユーザーのコメントの合間に、標準搭載されたアスキーアート（AA）や特殊な記号・顔文字を自動的に混入させて弾幕を賑やかにします。
⑤ リアルタイムUI調整:
   文字サイズ、スクロール速度、文字色、フチ（アウトライン）の色や太さを、OBSのプロパティパネルから即座に変更可能です。

------------------------------------------------------------------------
3. 設定パラメータ（プロパティ）仕様
------------------------------------------------------------------------
プラグインが保持する設定データ（obs_data_t）のキー、型、初期値および説明です。

| プロパティキー   | 型     | 初期値                 | 説明 |
| :---             | :---   | :---                   | :--- |
| file_path        | String | ""                     | コメントを読み込むテキストファイルの絶対パス |
| font             | Object | デフォルトフォント     | 描画に使用するフォント情報（face名、スタイル等） |
| font_size        | Int    | 36                     | 文字の大きさ (px) |
| speed            | Double | 250.0                  | コメントの標準スクロール速度 (px/sec) |
| color            | Int    | 0xFFFFFFFF (白)        | コメントの文字色 (BGR/RGB形式) |
| outline_color    | Int    | 0xFF000000 (黒)        | コメントのフチ（アウトライン）色 |
| outline_size     | Double | 10.0                   | 文字サイズに対するフチの太さの割合 (%) |
| max_comments     | Int    | 100                    | 画面上に同時に表示可能な最大コメント数 |
| interval         | Int    | 300                    | コメントを画面に追加する際の最小時間間隔 (ms) |
| nigayaka_prob    | Int    | 10                     | 賑やかし用AAや記号を自動混入する確率 (0〜100%) |
| enable_nigayaka  | Bool   | true                   | 賑やかしコメント全体の有効・無効オンオフ |
| direction        | String | "left_to_right"        | 流れる方向 ("left_to_right" または "right_to_left") |
| loop_play        | Bool   | true                   | デモ用のループ再生モード (falseで新規追記監視) |
| check_interval   | Int    | 1000                   | テキストファイルの変更を監視する周期 (ms) |

------------------------------------------------------------------------
4. 内部アーキテクチャ・主要関数
------------------------------------------------------------------------
ソースコードは `plugin-main.cpp` と `danmaku-source.cpp` で構成されています。

### 1) モジュールの初期化と登録 (plugin-main.cpp)
- `obs_module_load()`:
  GDI+を初期化（`GdiplusStartup`）し、ソース登録関数 `RegisterDanmakuSource()` を呼び出します。
- `obs_module_unload()`:
  GDI+をシャットダウン（`GdiplusShutdown`）します。

### 2) レンダリングとロジックの実装 (danmaku-source.cpp)
- `CreateTextTexture()`:
  GDI+を用いて、メモリ上の32bit ARGBビットマップにテキスト（アウトライン付き）をレンダリングします。
  一時的なファイル出力は一切行わず、ロックしたビットマップデータを `gs_texture_create` を用いて直接VRAM（テクスチャ）に転送（GS_BGRA形式）します。
- `PushComment()`:
  レーン（高さ = フォントサイズ * 1.4）の空き状況をスキャンします。流れる方向に応じた物理判定（衝突・追い越し予測）を行い、合格したレーンのいずれかにテクスチャ付きコメントを初期位置に配置します。
  【追い越し回避の計算式】
  新速度 $v_{new}$ が先行速度 $v_{last}$ より速い場合、先行位置 $x_{last}$ に対して：
  $\frac{x_{last}}{v_{new} - v_{last}} > \frac{canvas\_width - x_{last}}{v_{last}}$（左から右へ流す場合）
  が真であれば画面内で追いつかないと判断し、レーンへの投入を許可します。
- `danmaku_video_tick()`:
  - 毎フレーム経過時間（seconds）に応じて、アクティブコメントのX座標を `x += speed * seconds`（左から右の場合）で更新。
  - 画面外に完全にフェードアウトしたコメントの `gs_texture_t` を `obs_enter_graphics()` 空間で安全に破壊・メモリ解放します。
  - 監視間隔（`check_interval`）ごとにテキストファイルの最終更新時間（`last_write_time`）を監視。更新を検知した場合、ファイルを末尾まで再読み込みし、ループ再生または新規追記キューへの追加を行います。
  - 最小投入間隔（`interval`）が経過していれば、キューまたはループデータから新規コメントをレーンへ投入します。
- `danmaku_video_render()`:
  アクティブなコメントのテクスチャを、OBSの描画用ラッパー関数 `obs_source_draw` を呼び出して画面上の座標に直接レンダリングします。

5. 設計の堅牢性と安全性
------------------------------------------------------------------------
- **ゼロ値サイズガード**:
  OBSがソースのサイズ取得関数である `danmaku_get_width` と `danmaku_get_height` から `0` を受け取ると、拡大縮小や描画エンジン内部でゼロ除算が発生し、OBS Studio全体が即座にフリーズまたは強制終了します。
  本プラグインでは、キャンバスサイズが 0 以下の値になろうとした場合でも、常にデフォルト値（1920x1080）を返すように安全ガードが組み込まれています。
- **グラフィックス・コンテキストの遵守**:
  テクスチャの生成（`gs_texture_create`）や破棄（`gs_texture_destroy`）は、必ずOBSのグラフィックス実行スレッド空間を確保する `obs_enter_graphics()` / `obs_leave_graphics()` 内で限定して実行され、GPUクラッシュや黒画面化を防いでいます。
- **C++20規格対応の動的登録**:
  `obs_source_info` 構造体は、C++20で導入されたメンバ順序一致の制約（C7560エラー）を回避するため、静的イニシャライザではなく登録関数（`RegisterDanmakuSource`）による実行時の動的代入・登録方式を採用しています。
